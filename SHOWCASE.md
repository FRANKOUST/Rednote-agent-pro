# SHOWCASE

This repo now demonstrates a small-scale, provider-oriented content ops stack:

- collector provider: Scrapling search/detail wiring with fixture/live shells
- model provider: configurable OpenAI-compatible routing with schema validation
- sync provider: lark-cli/Feishu command execution with dry-run-first safety
- operator surfaces: REST + MCP + Web Console + diagnostics + health + audit logs
- release posture: manual-review-first, live publish disabled by default

Current engineering status: feature-complete for dry-run/fixture operation and awaiting only external credentials/login state for real validation.
